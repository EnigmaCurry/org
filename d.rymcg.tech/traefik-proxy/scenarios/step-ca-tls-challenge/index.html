Step-CA TLS challenge

This scenario covers a basic Traefik configuration for a public server (VPS). This uses Step-CA to sign off-grid TLS certificates. This scenario will use the TLS-ALPN-01 challenge type.

Step-CA is a self-hosted ACME provider, a root Certificate Authority. Step-CA is an alternative to Let’s Encrypt, but they do serve difference purposes:

  • Use Let’s Encrypt if you need “valid” certificates trusted by web browsers.
  • Use Step-CA if you want fully self-hosted PKI, and your own Certificate Authority. The certificates will not be validated by your clients unless they add your root CA public key to their OS or web browser trust store.
Warning

This scenario is only good for private websites and services, as the certificates are considered “self signed”, they will not be trusted by normal web browsers, unless you add it to the trust store.

Create a Step-CA server

Follow the chapter named Step-CA to create your own Step-CA powered ACME Certificate Authority.

ACME TLS-ALPN-01 challenge type

ACME challenges are a type of procedure that authenticates your server to Let’s Encrypt that you are the owner (controller) of your domain name system (DNS). This is important because it prevents other actors from requesting certificates for your domains.

The “TLS challenge” (TLS-ALPN-01), is one of two supported ACME Challenge types. TLS-ALPN-01 is the default type, which has some advantanges, and disadvantages.

  • It requires port 443 to be public. It cannot work when port 443 is blocked.
  • It only requires a valid DNS A record for the certificate domains to be pointing to the IP address of your server. For a public service, you would have needed to create this record anyway, so this isn’t really an extra requirement.
  • Compared to other supported challenge type, DNS-01, it is much easier and safer to configure, because it does not require the storage of a sensitive API token for your DNS provider.
  • A significant drawback is the limitation that it can only issue certificates for an explicit list of domain names. No wildcards (*.example.com) are allowed with TLS-ALPN-01. So you will need to revisit your traefik config every time you add a new application domain name.
  • Step-CA does not participate in Certificate Transparency logs.

Configure Traefik

Warning

If you want to reset the existing config back to the defaults, run:

[bash]: Run this on your workstation:
d make traefik clean
[bash]: Run this on your workstation:
d make traefik config

The configuration is driven by a text wizard menu system.

(stdout)
? Traefik config main menu:
> Create system user on Docker host
  Configure entrypoints (including dashboard)
  Configure Certificate Authorities (CA)
  Configure ACME (Let's Encrypt or Step-CA)
  Configure TLS certificates and domains (make certs)
  Configure middleware (including auth)
v Configure error page template
[↑↓ to move, enter to select, type to filter, ESC to cancel]

Use your arrow keys to select only the menu items this tutorial does not tell you to skip. Select the first one and press the Enter key. To go back, press the ESC key.

Once you go through all the options you need, it will have created and configured your .env_{CONTEXT} file for you.

Create system user on Docker host

This option will create a new username on the Docker host, called traefik. This is necessary to reserve a unique UID for traefik to run as on the system. Run this first before anything else.

Skip: Configure entrypoints

For this tutorial, you will use the default entrypoints, so you don’t need to configure anything.

  • websecure (TCP port 443, https)
  • web (TCP port 80, but only used as redirect to websecure)

Configure Certificate Authorities (CA)

You must configure Traefik instance to trust the root certificate of your Step-CA instance.

(stdout)
? How do you want to configure the list of trusted Certificate Authorities (CA)?
  Use the stock list, (alpine: ca-certificates ca-certificates-bundle).
> Use the stock list, plus add my own root Step-CA certificate.
  Delete the entire list, and add my own root Step-CA certificate.
  Delete the entire list.
  Cancel / Go back.
[↑↓ to move, enter to select, type to filter, ESC to cancel]

Enter the root URL of your Step-CA instance:

(stdout)
TRAEFIK_STEP_CA_ENDPOINT: Enter your Step-CA endpoint URL (eg. https://ca.example.com)

: https://ca.example.com

Configure ACME (Step-CA)

(stdout)
? Which ACME provider do you want to use?
  Let's Encrypt (ACME)
> Step-CA (ACME)
  Disable ACME
  Cancel / Go back

Choose Step-CA (ACME).

Choose TLS-ALPN-01 ACME challenge

(stdout)
? Which type of ACME challenge should be used?
> TLS-ALPN-01 (default for public servers, easy, but no wildcard certs)
  DNS-01 (requires API key, but good behind firewalls, and allows wildcard certs)
Tip

For your first install, choose TLS-ALPN-01, it is the easiest method to use for public servers.

If you want to use wildcard DNS records, or you need to setup a server behind a firewall, you must choose the more advanced method DNS-01, and setup your DNS platform for programmatic access with an API token.

Configure TLS certificates and domains

d.rymcg.tech uses explicit certificate requests configured centrally, on the traefik project:

(stdout)
? Configure Traefik TLS certificates
> Manage all certificates.
  Create a new certificate.
  Done / Go back

Manage all certificates

This will show you all the certificate requests that have been defined and allow you to manage each one. It will be blank to start out with.

Create a new certificate

Use this to define all the certificates you need for all your applications.

Each certificate needs a main name (CN), which should be the main domain name of the certificate.

(stdout)
Enter the main domain (CN) for this certificate

: prod.example.com
Info

Each certificate may also contain several other domain names, known as SANs (Subject Alternative Names). You can use this to list as many additional domain names that you want to be listed on the same certificate. The record for whoami is required for the next chapter, Whoami.

(stdout)
Now enter additional domains (SANS), one per line:
Enter a secondary domain (enter blank to skip)

: whoami.prod.example.com
Tip

Make sure that each domain name that you entered on the certificate has a valid DNS A record, pointing to your Docker server’s IP address.

Skip: Configure middleware (including auth)

No middleware is needed for this scenario.

Install Traefik

To install Traefik, you can simply choose the Reinstall Traefik option in the menu, or you can run that step all by itself from the command line at any time:

[bash]: Run this on your workstation:
d make traefik install

You should always reinstall Traefik after changing any configuration setting.
